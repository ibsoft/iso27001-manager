# Internationalization (i18n) — Multilingual Support

## Overview

The application uses **Flask-Babel 4.x** to provide full English and Greek (Ελληνικά) translations, with an extensible architecture that allows adding more languages by editing a single config dictionary and adding translation files.

### Supported Languages

| Code | Name (native)   |
|------|-----------------|
| `en` | English         |
| `el` | Ελληνικά        |

---

## Architecture

### 1. Flask-Babel Initialization

**File:** `app/__init__.py` (lines 38–48)

```python
from flask import request, session

LANGUAGES = {"en": "English", "el": "Ελληνικά"}

def get_locale():
    lang = session.get("lang")
    if lang in LANGUAGES:
        return lang
    return request.accept_languages.best_match(list(LANGUAGES.keys()), default="en")

babel.init_app(app, locale_selector=get_locale)
```

The locale selector function checks the user's `session["lang"]` first (set when they use the language switcher). If no session preference is found, it falls back to the browser's `Accept-Language` header via `request.accept_languages.best_match()`.

### 2. LANGUAGES Dictionary

The authoritative list of supported languages is defined in **two places** that must stay in sync:

| Location | Purpose |
|---|---|
| `config.py:45` | `Config.LANGUAGES` — used as app config |
| `app/__init__.py:40` | `LANGUAGES` local variable — injected into templates |

The `app/__init__.py` variable is passed to:
- `app.jinja_env.globals` — so `{{ LANGUAGES }}` is available in every Jinja2 template (line 91)
- A `@app.context_processor` — which also injects `current_lang` (lines 93–99)

```python
app.jinja_env.globals.update(now=dt_now.utcnow, LANGUAGES=LANGUAGES)

@app.context_processor
def inject_globals():
    from flask import session as _session
    return {
        "current_lang": _session.get("lang", "en"),
        "LANGUAGES": LANGUAGES,
    }
```

### 3. Language Switch Route

**File:** `app/routes/auth.py` (lines 17–22)

```python
@auth_bp.route("/language/<lang>")
def set_language(lang):
    from flask import session
    if lang in ("en", "el"):
        session["lang"] = lang
    return redirect(request.referrer or url_for("dashboard.index"))
```

This route is called when a user clicks a language link. It persists the choice in the session and redirects back to the previous page (or the dashboard if no referrer is available).

### 4. Template Integration

#### Base Template — Navbar Dropdown (`base.html:198–206`)

```html
<li class="nav-item dropdown">
  <a class="nav-link dropdown-toggle" href="#" id="langDropdown"
     data-bs-toggle="dropdown" aria-expanded="false">
    <i class="fas fa-globe me-1"></i>
    <span>{{ LANGUAGES.get(current_lang, 'English') }}</span>
  </a>
  <ul class="dropdown-menu dropdown-menu-end shadow-sm">
    {% for code, name in LANGUAGES.items() %}
    <li>
      <a class="dropdown-item {{ 'active' if current_lang == code }}"
         href="{{ url_for('auth.set_language', lang=code) }}">
        {{ name }}
      </a>
    </li>
    {% endfor %}
  </ul>
</li>
```

Shows a globe icon with the current language name. On click, displays all available languages. The active language is highlighted.

#### Standalone Pages — Macro (`_lang.html`)

Used on pages that do not extend `base.html` (e.g. the login page).

```html
{% macro lang_switch(current_lang) %}
{% for code, name in LANGUAGES.items() %}
{% if code != current_lang %}
<a href="{{ url_for('auth.set_language', lang=code) }}" class="small text-muted me-2">
  <i class="fas fa-globe me-1"></i>{{ name }}
</a>
{% endif %}
{% endfor %}
{% endmacro %}
```

Renders links for languages other than the current one.

#### Marking Translatable Strings in Templates

Two syntaxes are used:

- **Inline:** `{{ _('Sign In') }}`
- **Block:** `{% trans %}Some longer text{% endtrans %}`

Example from `login.html`:

```html
<label class="form-label fw-medium small text-muted">{{ _('Username') }}</label>
{{ form.username(class="form-control", placeholder=_('Enter your username')) }}
```

### 5. Python Route Files

Each route file imports and uses Flask-Babel's `gettext`:

```python
from flask_babel import gettext as _
```

Usage examples:

| Context | Pattern |
|---|---|
| **Flash messages** | `flash(_('User created successfully.'), 'success')` |
| **Page titles** | `render_template('users.html', title=_('User Management'))` |
| **Form choices** | `choices=[('value', _('Label'))]` |
| **Conditional strings** | `_("User activated.") if user.is_active else _("User deactivated.")` |

### 6. Translation Files

Generated files are stored at:

```
app/translations/
  el/
    LC_MESSAGES/
      messages.po    # editable Greek translation source
      messages.mo    # compiled binary (loaded at runtime)
messages.pot          # translation template (regenerated from source)
```

### 7. babel.cfg

**File:** `babel.cfg`

```ini
[python: **.py]
[jinja2: **/templates/**.html]
```

Tells `pybabel extract` which file types to scan for translatable strings.

### 8. Configuration

**File:** `config.py:45–46`

```python
LANGUAGES = {"en": "English", "el": "Ελληνικά"}
BABEL_DEFAULT_LOCALE = "en"
```

`BABEL_DEFAULT_LOCALE` is the fallback when no user preference or browser match is found.

## Workflow: Adding a New Language

### Step 1 — Register the language

Edit `config.py` and `app/__init__.py`:

```python
LANGUAGES = {
    "en": "English",
    "el": "Ελληνικά",
    "fr": "Français",       # new
}
```

### Step 2 — Update the route guard

Edit `app/routes/auth.py:20`:

```python
if lang in ("en", "el", "fr"):
```

> **Alternative:** Replace the hardcoded tuple with `app.config["LANGUAGES"].keys()` for a fully dynamic check.

### Step 3 — Regenerate the template

```bash
source venv/bin/activate
pybabel extract -F babel.cfg -o messages.pot app/ config.py run.py wsgi.py
```

### Step 4 — Initialize the new language

```bash
pybabel init -i messages.pot -d app/translations -l fr
```

This creates `app/translations/fr/LC_MESSAGES/messages.po`.

### Step 5 — Translate

Edit the new `.po` file and fill in each `msgstr ""` with the French translation:

```po
msgid "Sign In"
msgstr "Connexion"
```

### Step 6 — Compile

```bash
pybabel compile -d app/translations
```

### Step 7 — Verify

```bash
# Quick smoke test
python -c "
import os
os.environ['FLASK_CONFIG'] = 'development'
from app import create_app
app = create_app('development')
with app.test_request_context():
    from flask import session
    session['lang'] = 'fr'
    from flask_babel import refresh, gettext as _
    refresh()
    print(_('Sign In'))    # should print 'Connexion'
"
```

## Workflow: Updating Translations After Code Changes

After adding/changing translatable strings in templates or Python files:

```bash
source venv/bin/activate

# 1. Extract new strings
pybabel extract -F babel.cfg -o messages.pot app/ config.py run.py wsgi.py

# 2. Merge into existing .po files (preserves existing translations)
pybabel update -i messages.pot -d app/translations

# 3. Edit the .po files to translate new strings (msgstr "" entries)

# 4. Recompile
pybabel compile -d app/translations
```

The `update` command is preferred over `init` because it preserves existing translations and only adds new entries.

## Fallback Behavior

If a string is not found in the selected language's `.mo` file, Flask-Babel automatically falls back to the `BABEL_DEFAULT_LOCALE` (English). This means:
- Partial translations are safe — untranslated strings show in English.
- You can roll out translations incrementally per language.
